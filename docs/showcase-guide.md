# Showcase Guide

## Suggested screenshots

Capture these in order for a GitHub README, portfolio site, or LinkedIn post:

1. Landing page hero
2. Upload flow with drag-and-drop area
3. Completed analysis dashboard top section
4. Score breakdown chart and keyword groups
5. Suggestions panel
6. Saved analysis history page
7. Shareable results page

## Demo checklist

- Have at least one completed analysis session in the database
- Have one session with a strong score and one with weaker alignment
- Prepare a resume file that parses cleanly
- Prepare a realistic job description with recognizable tools and role language
- Verify that the report download route works before recording

## Best story to tell

- show the upload flow first to establish the product entry point
- show the dashboard second because it is the visual centerpiece
- show the history page third to prove persistence and product depth
- finish with share/report actions to emphasize polish

## What recruiters should notice

- real file handling, not just form text boxes
- modular full-stack architecture
- transparent scoring logic
- clean data persistence story
- premium UI quality

## README/GitHub tips

- Pin 1-2 strong screenshots near the top of the repo once you capture them
- Keep the project pitch to one sentence before diving into technical detail
- Put setup steps and demo value both in the README so the repo works for engineers and recruiters
